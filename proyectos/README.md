# Proyectos

Esta carpeta contiene la documentación de todos los proyectos existentes de la empresa.

## Estructura

Cada proyecto tiene su propia carpeta con esta estructura:

```
nombre-proyecto/
├── README.md           # Documentación principal del proyecto
├── ARCHITECTURE.md     # (Opcional) Arquitectura detallada
├── API.md             # (Opcional) Documentación de API
└── ...                # Otros documentos relevantes
```

Si prefieres un solo archivo por proyecto, simplemente usa:
```
nombre-proyecto.md
```

## Información Clave por Proyecto

Cada proyecto debe documentar:
- Descripción y propósito
- URL del repositorio GitHub
- Tecnologías utilizadas
- URLs de entornos (prod, staging, dev)
- Instrucciones de configuración local
- Proceso de deployment
- Equipo responsable

## Cómo Agregar un Proyecto

1. Copia `TEMPLATE.md` como `nombre-proyecto.md` o crea carpeta `nombre-proyecto/`
2. Completa toda la información relevante
3. Actualiza esta lista de proyectos actuales
4. Registra el cambio en `/CHANGELOG.md`

## Estados de Proyectos

- **Desarrollo**: En construcción activa
- **Producción**: En uso por clientes/usuarios
- **Mantenimiento**: Recibe actualizaciones periódicas
